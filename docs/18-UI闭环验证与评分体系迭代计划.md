# UI闭环验证与评分体系迭代计划（公司执行版）

## 1. 目标

把当前“筛选后导出到 Excel 手工复评”的流程，升级为“UI 内持续跟踪验证 + 自动反哺参数”的闭环系统。

优先满足两件事：

1. 在免费数据源前提下，持续稳定获取复评所需数据
2. 用真实后验表现迭代评分体系，追求可交易、可持续的盈利能力

---

## 2. 总体闭环流程

统一成 5 个步骤：

1. 当日筛选：生成候选股列表
2. 保存实验批次（run）：固化当时参数与候选快照
3. 后续跟踪：T+1/T+2/T+3/T+5/T+10 自动补行情
4. 复评分析：查看单票与组合真实表现（收益+回撤）
5. 策略反馈：比较权重组合，给出下一版参数建议

---

## 3. UI 页面规划

## 3.1 页面A：筛选与出池（现有页面增强）

新增能力：

1. 每次运行筛选后自动生成 `run_id`
2. 固化运行参数（权重、阈值、top_n、采集范围、数据源健康信息）
3. 自动保存候选快照（用于后续追溯）

页面提示：

- 本次 run_id
- 当前数据源健康度
- 当前 run 是否已纳入持续复评

## 3.2 页面B：复评中心（新增）

核心交互：

1. 选择 run_id
2. 点击“更新复评数据”
3. 展示：
   - 单票表现表（每只票多周期收益与回撤）
   - 组合统计卡片（平均收益、胜率、盈亏比、最大回撤）
   - 分层表现（Top3/Top5/Top10 对比）

建议显示字段：

- `ret_close_t1/t2/t3/t5/t10`
- `max_drawdown_t3/t5/t10`
- `max_runup_t3/t5/t10`
- `hit_stop_loss` / `hit_take_profit`
- `status`（待更新/已完成/数据不足）

## 3.3 页面C：参数实验室（新增）

核心功能：

1. 选择历史 run 时间范围
2. 自动跑多组权重组合
3. 对比真实绩效并输出“推荐权重区间”

显示结果：

- 权重组合排行榜（按风险调整收益）
- 稳定性指标（不同阶段表现波动）
- 一键应用到筛选页（填充滑块默认值）

## 4. 数据结构设计（MVP）

先用本地 CSV，后续可迁移 SQLite。

1. `data/runs.csv`
   - run 元信息：`run_id`、创建时间、参数、数据源状态

2. `data/run_candidates/{run_id}.csv`
   - 当日候选快照：symbol/name/theme/四项子分/total_score

3. `data/run_evaluations/{run_id}.csv`
   - 复评结果：多周期收益、回撤、命中标签、状态

4. `data/strategy_report.csv`
   - 每个 run 的组合摘要（用于周报和参数实验）

建议文件：

- `scripts/experiment_weights.py`

能力要求：

2. 生成候选权重集合（网格或随机）

4. 输出排行榜与推荐区间

## 6. 数据源健康度评分 + 自动降级策略

健康度评分用于告诉你“今天这份数据还能不能信”。

评分建议（0~100）：

1. 字段覆盖率（行业/换手/量比）
2. 新鲜度（snapshot 时间）
3. 采集成功率（近 N 次）
4. 跨源偏差（可选，后续加）

自动降级策略：

1. 主源失败 -> 备源
2. 备源失败 -> 缓存
3. 缓存也不可用 -> 明确报错，不隐式继续

UI 必须展示：

- 当前使用源
- 是否处于降级模式
- 本次健康分及主要扣分项

---

## 7. 反哺评价体系的方法（避免只看“高分”）

每周固定看 4 类指标：

1. 收益：平均收益、中位数收益
2. 风险：最大回撤、波动
3. 质量：胜率、盈亏比
4. 稳定：不同周之间表现方差

参数调整规则：

1. 每周最多调一次
2. 每次只改 1-2 个参数
3. 连续两周同向验证再固化
4. 不追求单次最优，优先稳健区间

## 8. 分阶段执行计划（明天起可落地）

## Phase 1（MVP，1~2天）

1. 生成 run_id + 保存 run/candidates
2. 新增“复评中心”页（手动更新按钮）
3. 升级多周期复评脚本（至少支持 T+1/T+3/T+5）

验收标准：

- 能在 UI 选 run 并看到复评表
- 不需要 Excel 手工搬运数据

## Phase 2（2~3天）

1. 增加回撤/冲高回落统计
2. 增加组合绩效卡片和分层分析
3. 增加“更新全部未完成 run”

验收标准：

- run 在验证窗口内可持续自动更新
- 组合表现可用于周度复盘

## Phase 3（2~3天）

1. 新增权重实验脚本
2. 新增参数实验室页面
3. 产出推荐权重区间并支持应用

验收标准：

- 可自动比较多组权重真实表现
- 给出下一版参数建议，不靠主观拍脑袋

## 9. 明天到公司后的开工顺序（按步骤）

## 10. Done Definition（完成判定）

满足以下条件才算“闭环初版完成”：

1. 每次筛选都有 run_id 且可追溯参数
2. 同一批候选可在后续多天持续复评
3. UI 内可直接看到单票与组合真实表现
4. 参数调整有后验证据，不再依赖主观感觉
