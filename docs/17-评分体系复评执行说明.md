# 评分体系复评执行说明

## 1. 目标

验证“当前评分体系是否真的能选出符合你交易思想的标的”，而不是只看当下排序是否漂亮。

## 2. 每日执行（3步）

1. 在 UI 运行筛选，得到 `candidates_*.csv`
2. 系统自动生成 `candidates_review_template_*.csv`
3. 次日收盘后运行：

```bash
python scripts/evaluate_candidates_next_day.py --input output/candidates_*.csv
```

生成 `*_evaluated.csv`，包含次日开高低收及次日收盘相对当日收盘涨跌幅。

## 3. 每周复评（建议周日）

统计最近一周候选池：

1. TopN 的次日平均涨跌幅
2. Top3 与 Top10 的差异（看排序是否有区分度）
3. 不同数据质量水平下的表现差异（行业/换手/量比覆盖率）
4. 命中“你的主观强势票”的比例

## 4. 调整规则（避免过拟合）

- 每周最多调整一次参数
- 每次只改 1-2 个权重或阈值
- 连续两周同向验证后再固化规则

## 5. 关键判断问题

1. 高分票是否更容易在次日延续强势？
2. 数据覆盖率差时，模型表现是否显著下滑？
3. 哪个因子在你风格中最有解释力（题材/联动/个股/资金）？
4. 当前“行业代理题材”是否足够，是否必须升级到概念题材？
